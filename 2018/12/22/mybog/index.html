react基础整理

发表于 2018-12-22

一、react优势

  • 虚拟DOM，HTML写在js里
  • 一切皆组件，支持自定义
  • 单向数据流，跨平台运行

二、组件Component

  • 创建组件的三种方式：
    1. es6创建 ：React.component
    2. es5创建 : React.createClass
    3. 函数式定义的无状态组件：无状态的组件形式上表现为一个只待油render方法的组件类，通过函数形式或者箭头函数，该组件无state状态
      建议选择第一个和第三个
  • 组件的特点
    1. 组件中只能有一个根结点
    2. 组件首字母必须大写，否则会被解析成html标签
    3. 具有可重用、可组合、可维护特征
    4. 对应的标签语法和html标签用法一致，可加入任意属性

四、Props、State对比

  • 相同点
    1. 都是用于描述组件状态的
    2. 都可以改变，改变都会触发组件的重新渲染
  • 不同点
    1. Props是由外部传入的，是父组件传递给子组件的数据流。
    2. State是内部定义的，代表组件的内部状态。在内部改变与外部组件没有直接联系。
    3. Props通常在组件外部发生变化，在内部保持不变。
    4. 一个组件不能改变自身的props, 但要负责设置子组件的 props

五、箭头函数

+ es6之前需要在调用函数的地方使用bind绑定 this。
render(){ return <button onClick={this.handleClick.bind(this)} >{this.props.name}</button>; }。
ES6之后使用箭头函数就不用了，因为es6的箭头函数绑定定义时所在的作用域

React提供了一个获取组件的方式Refs。能用State或者Props的不要用Refs

    • 触发强制动画
    • 处理焦点、文本选择或媒体控制
