react基础整理

发表于 2018-11-25

一、react优势

  • 虚拟DOM，HTML写在js里
  • 一切皆组件，支持自定义
  • 单向数据流，实现了单项响应的数据流，减少了重复代码，比传统的数据绑定更简单
  • jsx: javascript语法的扩展。

    二、组件Component

  • 创建组件的三种方式：
    1. es6创建 ：React.component
    2. es5创建 : React.createClass
    3. 函数式定义的无状态组件：无状态的组件形式上表现为一个只待油render方法的组件类，通过函数形式或者箭头函数，该组件无state状态
      建议选择第一个和第三个
  • 组件的特点
    1. 组件中只能有一个根结点
    2. 组件首字母必须大写，否则会被解析成html标签
    3. 具有可重用、可组合、可维护特征
    4. 对应的标签语法和html标签用法一致，可加入任意属性

四、Props、State对比

  • 相同点
    1. 都是用于描述组件状态的
    2. 都可以改变，改变都会触发组件的重新渲染
  • 不同点
    1. Props是组件的配置，是由外部传入的，是父组件传递给子组件的数据流。不仅仅是数据，回调函数也可以通过props传递
    2. State是是一种数据结构，由内部定义的，代表组件的内部状态。在内部改变，与外部组件没有直接联系。
    3. 一个组件不能改变自身的props, 但要负责设置子组件的 props

五、箭头函数

  • es6之前需要在调用函数的地方使用bind绑定 this。
    render(){ return ; }。
    ES6之后使用箭头函数就不用了，因为es6的箭头函数绑定定义时所在的作用域

六、JSX

  • React使用JSX来代替常规的javascript
  • JSX执行更快，因为他在编译js代码后进行了优化
  • 类型安全，在编译过程中就能发现错误。使用jsx编写模板更加简单快速

  • 可以在jsx中使用javascript表达式，写在花括号{}中.

  • 推荐使用内联样式

七、 获取真实的dom节点refs

  • React提供了一个获取DOM节点或react组件的方式引用–Refs。
  • 避免使用String类型的Refs和内联的ref回调，Refs回调是React所推荐的
  • 使用场景：
    • 触发强制动画
    • 处理焦点、文本选择或媒体控制
    • 集成第三方dom库

八、类组件(Class component)和函数式组件(Functional component)的区别

  • 类组件允许你使用更多额外的功能，如组件自身的状态和生命周期钩子，也能使组件直接访问store并维持状态
  • 函数式组件仅是接收 props，并将组件自身渲染到页面时，该组件就是一个 ‘无状态组件(stateless component)’，可以使用一个纯函数来创建这样的组件。这种组件也被称为哑组件(dumb components)或展示组件
