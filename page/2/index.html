H5缓存机制

发表于 2018-01-24

目前H5一共有6种缓存机制

一、浏览器缓存机制

指通过Http协议头里的Cache-Control(Expires)和last-Modified(Etag)等字段控制缓存的机制
  • Cache-Control 用于控制文本在本地缓存有效时长
  • Last-Modified 标识文件在服务器上的最新更新时间
  • Cache-Control 通常与 Last-Modified 一起使用。一个用于控制缓存有效时间，一个在缓存失效后，向服务查询是否有更新
  • Cache-Control 还有一个同功能的字段：Expires。Expires 的值一个绝对的时间点，如：Expires: Thu, 10 Nov 2015 08:45:11 GMT，表示在这个时间点之前，缓存都是有效的。
  • Etag 也是和 Last-Modified 一样，对文件进行标识的字段。不同的是，Etag 的取值是一个对文件进行标识的特征字串

二、Dom Storgage(Web Storage)存储缓存，缓存分为localStorage、sessionStorage、cookie等

  • localStorage： 永久性的本地存储，存储在客户端的浏览器上，不会过期，直到手动删除，采用键值对的方式进行存储，存储方式为字符串
    API有：
    1. localStorage.setItem()存储数据
    2. localStorage.getItem()获取数据，使用localStorage.valueOf()获取全部数据
    3. localStorage.removeItem(key) 删除数据
    4. localStorage.clear()清空全部数据，localStorage.length 获取本地存储数据数量
    5. localStorage.key(N)获取第N个数据的key键值
    • SessionStorage: 用于本地存储一个会话中的数据，只有在同一个会话中的页面才能访问，结束回话窗口关闭后数据就会消失，属于会话级别的存储

http协议攻略

发表于 2018-01-05

一、http请求报文

  • 一个HTTP请求报文由4个部分组成

    1. 请求行,由请求方法字段、URL字段和HTTP协议版本字段3个字段组成，它们用空格分隔:HTTP协议的请求方法有GET、POST、HEAD、PUT、DELETE、OPTIONS、TRACE、CONNECT.GET方法要求服务器将URL定位的资源放在响应报文的数据部分，回送给客户端
    2. 请求头部:请求头部由关键字/值对组成，每行一对，关键字和值用英文冒号“:”分隔。请求头部通知服务器有关于客户端请求的信息，典型的请求头有： User-Agent：产生请求的浏览器类型。
      Accept：客户端可识别的内容类型列表。
      Host：请求的主机名，允许多个域名同处一个IP地址，即虚拟主机。
    3. 空行:最后一个请求头之后是一个空行，发送回车符和换行符，通知服务器以下不再有请求头。
    4. 请求数据:请求数据不在GET方法中使用，而是在POST方法中使用。POST方法适用于需要客户填写表单的场合。与请求数据相关的最常使用的请求头是Content-Type和Content-Length。

    二、http响应报文

    • HTTP响应也由三个部分组成，分别是：
      http响应状态码:
      1xx：指示信息–表示请求已接收，继续处理。

flex

发表于 2018-01-03

一、flex弹性布局盒模型

  • 2009年w3c提出，给予容器控制内部元素高度和宽度的能力
  • 已支持chrome、opera、firefox、safari、ie等，其中在webkit内核的浏览器中使用时，必须加上-webkit-前缀

二、容器属性介绍

  • flex-direction:决定主轴的方向，即项目排列的方向，有四个可能的值：row(默认)|row-reverse|column|column-reverse
  1. row:主轴为水平方向，项目沿主轴从左至右排列
  2. column：主轴为竖直方向，项目沿主轴从上至下排列
  3. row-reverse：主轴水平，项目从右至左排列，与row反向
  4. column-reverse：主轴竖直，项目从下至上排列，与column反向
  • flex-wrap (换行): nowrap(不换行默认), wrap, wrap-reverse(换行且往上一行换)
  • flex-flow (以上两者的简写): flex-direction || flex-wrap

框架学习之webpack

发表于 2017-12-29

一、概念

  • webpack 是一个模块打包器，可以递归打包项目中的所有模块，最终生成打包后的文件支持code-spliting、模块化(AMD,ESM,COmmonJS)、全局分析

    • 模块化AMD和CMD,模块加载器require.js和SeaJS，RequireJS 遵循 AMD（异步模块定义）规范，Sea.js 遵循 CMD （通用模块定义）规范。规范的不同，导致了两者 API 不同。Sea.js 更贴近 CommonJS Modules/1.1 和 Node Modules 规范。
    • 不仅实现了js文件的异步加载，还管理了模块之间的倚赖性
    • 对于依赖的模块，AMD 是提前执行，CMD 是延迟执行。不过 RequireJS 从 2.0 开始，也改成可以延迟执行（根据写法不同，处理方式不同）。CMD 推崇 as lazy as possible.
    • CMD 推崇依赖就近，AMD 推崇依赖前置。

  • 安装npm install webpack webpack-dev-server –save-dev

  • bundle是由webpack打包出来的文件，chunk是在webpack进行模块的依赖分析时，分割出来的代码块。module是开发中的单个模块

发表于 2017-12-27

一、js数据类型

  • javaScritp的数据类型有：数值类型、字符串类型、布尔类型、null、undefined、对象(数组、正则表达式、日期、函数)，大致分成两种：基本数据类型和引用数据类型
  • 基本数据类型保存在栈内存，引用类型保存在堆内存中。根本原因在于保存在栈内存的必须是大小固定的数据，引用类型的大小不固定，只能保存在堆内存中，但是可以把它的地址写在栈内存中以供我们访问

二、浅拷贝
