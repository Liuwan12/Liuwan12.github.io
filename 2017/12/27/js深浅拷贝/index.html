js深浅拷贝

发表于 2017-12-27

一、js数据类型

  • javaScritp的数据类型有：数值类型、字符串类型、布尔类型、null、undefined、对象(数组、正则表达式、日期、函数)，大致分成两种：基本数据类型和引用数据类型
  • 基本数据类型保存在栈内存，引用类型保存在堆内存中。根本原因在于保存在栈内存的必须是大小固定的数据，引用类型的大小不固定，只能保存在堆内存中，但是可以把它的地址写在栈内存中以供我们访问

二、浅拷贝

  • 只是拷贝基本的数据类型，浅拷贝只复制指向某个对象的指针，而不复制对象本身，新旧对象还是共享同一块内存

三、深拷贝

  • 对目标的完全拷贝,创造一个一模一样的对象，修改新对象不会改到原对象
  • 实现方法
    1. 利用 JSON 对象中的 parse 和 stringify (js值和JSon字符串相互转换)
      • JSON.stringify 是将一个 JavaScript 值转成一个 JSON 字符串
      • JSON.parse 是将一个 JSON 字符串转成一个 JavaScript 值或对象
      • undefined、function、symbol 会在转换过程中被忽略，所以对象中含有一个函数时（很常见），就不能用这个方法进行深拷贝
    2. 利用递归来实现每一层都重新创建对象并赋值

递归实现对象深拷贝

四、concat和slice

  • 在 JavaScript 中，数组有两个方法 concat和slice是可以实现对原数组的拷贝的，这两个方法都不会修改原数组，而是返回一个修改后的新数组。
  • concat和slice只是对数组的第一层进行深拷贝

五、Object.assign()和…展开运算符

  • Object.assign() 拷贝的是属性值。假如源对象的属性值是一个指向对象的引用，它也只拷贝那个引用值。
  • … 实现的是对象第一层的深拷贝。后面的只是拷贝的引用值。
  • 赋值运算符 = 实现的是浅拷贝，只拷贝对象的引用值
